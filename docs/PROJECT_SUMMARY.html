Minhnt Charger: ESP32 Gateway V2.2

Báo cáo tổng hợp kiến trúc hệ thống, Firmware và đặc tả nghiệp vụ

🚀 System Validation: Đã phát hiện và sửa chữa thành công BUG-2 (GPIO Pinout UART1) & BUG-8 (COM1 Disabled) trong Firmware. Sẵn sàng tiến hành idf.py flash.

Về Phần cứng

  • Vi điều khiển: ESP32-WROVER-B (8MB Flash, 4MB PSRAM).
  • Mạng Chính: IC PHY LAN8742A (Ethernet RMII) chống nhiễu (EMI). Chỉ bật WiFi AP khi nhấn nút GPIO34 (5 giây).
  • Giao tiếp Modbus (Bản DEV): Giao tiếp STM32 Slave thông qua RS485 Cổng 2 (COM2) trên chân dự phòng `GPIO32/33`. Tránh dùng Cổng 1 do lỗi thiết kế IC ăn chung vào UART0 Console.
  • Phụ trợ: DS1307 RTC, Hệ thống đèn trạng thái LED cảnh báo, Nút nhấn cứng.

Về Kiến trúc Phần mềm

  • RTOS Tasks: 7 FreeRTOS tasks đồng thời. Modbus được pin trên Core 1 để tránh độ trễ từ Network Stack (trên Core 0).
  • Memory Layout: Flash 8MB quy hoạch rất chuẩn xác với phân vùng storage và OTA kép ota_0 / ota_1.
  • Event Driven: Core hệ thống sử dụng esp_event_loop thay vì polling liên tục, giúp truyền tin nhắn cross-task nhẹ nhàng.
  • LittleFS: Quản lý bộ nhớ non-volatile cho UI Dashboard (phân vùng www) và queue rớt mạng (phân vùng storage).

Khối tính năng Trọng điểm (Store-And-Forward)

Dự án tích hợp một module lưu trữ xuất sắc để chịu đựng đứt gãy kết nối mạng:

  • Khi Event hoặc Telemetry không thể nổ MQTT vì mất mạng ethernet, bản tin được đóng block JSON nhị phân 64 Bytes lưu vào Flash LittleFS.
  • Dung lượng queue khổng lồ: 28,672 block, chứa khoảng 24h hoạt động của 10 thiết bị.
  • Replay liên tục tuần tự đẩy lên lại mạng theo thứ tự thời gian ngay khi LAN8742A báo kết nối. Có FIFO trim khi đầy bộ nhớ.

Luồng giao tiếp (Modbus & MQTT)

Nhiệm vụ Mô tả chức năng chi tiết Bảo mật & Tối ưu
Modbus RTU Polling Quét mã 0x04 trên 45 thanh ghi Input định kỳ mỗi 1 giây (V, A, W, kWh). Gửi mã Heartbeat 0x06 mỗi 3s. Loại bỏ Slave rớt để duy trì frame time vòng quay. Dynamic limit dòng.
MQTT Payload V3.4 Kết nối Backend qua TLS Port 8883. Cung cấp 7 loại payload (gw_status, tlm, status, evt, cmd, cmd_ack, ota_progress). NVS chứng chỉ CA gốc bảo vệ man-in-the-middle.
Local Web Dashboard HTTP Server chạy SPA, Websocket /ws xuất trạng thái Realtime cho người cài đặt local. Xác thực qua Cookie Session với Admin Password lưu ở NVS.
